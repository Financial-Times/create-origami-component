This folder structure reflects the **current** working folder structure for all Origami components, and is what gets generated by `obt init`.
```
.
├── MIGRATION.md
   // Migration guides for each major version
├── README.md
   // Description and usage instructions for the component
├── bower.json
   // Component dependencies
├── demos
│   └── src
│       ├── demo.js
            // demo-specific JavaScript (includes initialising component JS, can handle other events revolving around interacting with the demo)
│       ├── demo.mustache
           // markup template for demos, there can be various mustache files depending on the demo requirements
│       ├── demo.scss
           // demo-specific SCSS (includes initialising component styling)
│       └── pa11y.mustache
           // for a11y testing, should contain a version of every variation of the demos to test against
├── main.js
   // the entry file for a component's JavaScript, should fetch files from src/js
├── main.scss
   // the entry file for a component's SCSS, should fetch files from src/scss and any dependency's SCSS, should contain primary mixin.
├── origami.json
   // Origami manifest, names components, indicates what JS/SCSS/mustache files should be used to build demos, indicates component information, status, support and brands, identifies Origami components for the origrami-repo-data service
├── src
│   ├── js
│   │   └── [COMPONENT_NAME].js
           // Component class, can be multiple files / classes
│   └── scss
│       ├── _functions.scss
           // SCSS functions used across component styles
│       ├── _mixins.scss
           // SCSS mixins used across component styles
│       └── _variables.scss
           // Component-specific variables, including silent-mode
└── test
    ├── helpers
    │   └── fixtures.js
            // Fixtures for JS tests, usually imitates HTML generated in demos
    ├── scss
        // SCSS test specific folder, True requires this path (tests/scss) to run SCSS tests
    │   └── _index.test.scss
            // test entry file, must be called `_index.test.scss`
    └── [COMPONENT_NAME].test.js
        // Tests for JS in parallel src/js files
```
